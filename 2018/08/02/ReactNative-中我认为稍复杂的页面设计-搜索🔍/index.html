ReactNative 中我认为稍复杂的页面设计-搜索

发表于 2018-08-02 | | 本文阅读量
ReactNative中，大部分逻辑还是比较简单和易操作的。从着手开发过三个已上线的项目和无数demo来看，其中稍比较复杂的页面还是有那么几个的。个人感觉，当时这些问题好像很难解决，但是由于我的认真琢磨（ShuiLeYiJiao）之后，我还是找到解决方式。就是现在返回头看自己当初写的代码，还是觉得解决策略上是值得肯定的（笔者比较自恋，此处有不适者请见谅）。所以，决定把之前那些困扰我的稍复杂的页面。总结一下，一来自己复习总结，二来给后来者提供一个参考思路。

ReactNative中一个搜索页面的设计与实现

搜索功能是绝大多数App应有的一个功能，以方便快捷的方式在App内搜索自己想要查找的内容，这无疑是在用户体验上有着很大的提升。

  1. 我们在日常的最常用的搜索就是在输入框中输入自己检索的关键字,然后检索出的内容按列表排列供我们选择。
  2. 当然，可能一些app会比较了解用户习惯，默认会提供一些热词，我们直接点击这些热词就可以完成检索。
  3. 还有，比如说我们只是关心和自己有关的信息，我们总是搜索一个词条，下次我们不喜欢干同样重复的事情，那么App就会记录这些搜索记录，免去你的重复工作。
  4. 再有，比如你通过某一个关键词找到一组信息，那么你想要的是只在某个区间内的。那么我们可以通过再筛选来尽量精细我们要找的内容。

一、实现的目标

  1. 布局
    • 输入框输入检索内容
    • 筛选提示
    • 可清空输入内容按钮
    • 搜索按钮
    • 提示列表/检索结果列表
    • 热门词汇
    • 搜索历史列表倒序（当有搜索历史时）
    • 清除搜索历史
    • 无内容页面（保护）
  1. 交互
    • 当输入框为空的时候，展示为：搜索框+提示热词+搜索历史列表（如果有）
    • 当在输入框输入检索内容时返回提示列表/检索结果列表
    • 点击搜索按钮或者软键盘上的搜索按钮时刷新搜索（输入框不为空）
    • 点击筛选提示出现可供选择的筛选条件
    • 点击搜索热词，输入框填充该热词并开始搜索
    • 点击清空搜索内容时，界面回到初始状态
    • 点击某条搜索历史，输入框填充该内容并开始搜索
    • 点击清除搜索历史，搜索历史项置空
