JavaScript的一些不常见操作

  1. toString()的操作
  1. 对于typeof和instanceof使用的弊端

  • 可以通过Object.prototype.toString.call(XXX) ===’[object Object]’来避免这种弊端。
  1. NaN的一些使用
  1. 神奇的console.log
  1. Instanceof 和typeof 的实现原理
  • typeof 一般用于判断一个变量的类型，可以用typeof来判断 number，string，object，boolean，function，undefined，symbol这七种类型。
  • 由于js底层的存储机器码不同（对象：000，null的所有机器码均为0），所以出现typeof对于null的判断出现和object相同的结果。
  • 可以使用Object.prototype.toString.call()方法来判断一个变量的类型
  • instanceof来判断对象的具体类型，也就是说判断一个实例是否属于某种类型
  • instanceof也可以判断一个实例是否是其父类型或者祖先类型的实例
  1. 属性描述对象
    1. value ——表示属性值，默认值为 undefined
    2. writable ——表示属性是否可写，默认值为 true
    3. enumerable ——表示属性是否可以遍历，默认值为 true
    4. configurable ——表示属性是否可以配置，默认值为 true
    5. get ——表示属性的取值函数getter,默认值为 undefined
    6. set ——表示属性的存值函数setter，默认值为 undefined

  • 方法 1：Object.getOwnPropertyDescriptor() —— 获取属性的对象

  • 方法 2：Object.getOwnPropertyNames() ——获取属性的属性名，返回一个数组【不管该属性是否可以遍历】

  • 方法 3：Object.defineProperty(object，propertyName, attributesObject)

    • object：属性所在对象（可以是新建对象）
    • propertyName：属性名（是一个字符串）
    • attributesObject：属性描述对象，默认writable、configurable、enumerable为false

  • 方法 4：Object.defineProperties(object，propertyName, attributesObject)

    • 如果对象中存在取值函数get（或者存值函数set），就不能将writable属性设置为true，或者同时定义value属性，否则会报错

  • 方法 5：Object.prototype.propertylsEnumerable() ——判断某个属性是否可以遍历

    • 继承自原型对象的属性是不可以遍历的

  • 原属性 value：

    • 只要writable和configurable有一个为true，就允许改动。
    • configurable为false时，直接目标属性赋值，不报错，但不会成功。

  • 原属性 configurable：

    • 决定了属性目标是否可以被删除（delete）

  • 存取器

    • 存取器分别为存值函数setter，使用属性描述对象的set属性；取值函数getter，使用属性描述对象的get属性
    • 一旦对目标属性定义了存取器，那么存取的时候，都将执行对应的函数。
    • 取值函数getter不能接受参数，存值函数setter只能接受一个参数（即属性的值）。

  • 控制对象状态

    • Object.preventExtensions ——可以使得一个对象无法再添加新的属性
    • Object.isExtensible —— 检查一个对象是否使用了Object.preventExtensions方法。也就是说，检查是否可以为一个对象添加属性。
    • Object.seal ——可以使得一个对象即无法添加新属性，也无法删除旧属性

    Object.seal 的实质是把属性描述对象的configurable属性设置成了false，因此描述对象不能再改变了

    • Object.isSealed ——用于检查一个对象是否使用了Object.seal方法
    • Object.freeze ——使得一个对象无法添加新属性、无法删除旧属性、也无法改变属性的值，使得这个对象实际上变成了常亮
    • Object.isFrozen ——检查一个对象是否使用了Object.freeze方法
  1. 构造函数和New命令

    构造函数的特点：

    • 函数体内部使用了this关键字，代表了所要生成的对象实例
    • 生成对象的时候，必须使用new命令
  • new 命令原理：
    • 创建一个空对象，作为将要返回的对象实例
    • 将这个空对象的原型，指向构造函数的prototype属性
    • 将这个空对象赋值给函数内部的this关键字
    • 开始执行构造函数内部的代码
  1. 定时器

    JavaScript提供定时执行代码的功能，叫做定时器timer,主要由setTimeout()和setInterval()这俩个函数来完成。

  • setTimeout（）
    • 用来指定某个函数或者某段代码，在多少毫秒之后执行。它返回一个整数，表示定时器的编号，以后可以用来取消这个定时器。
    • setTimeout的第一个参数可以是fun|code,第二个参数是delay表示延迟执行的时间，单位为毫秒。如果第二个参数省略，则默认为0。
    • setTimeout除了俩个基本的参数，还允许加入更多的参数。它们将依次传入推迟执行的函数（回调函数）。
  • setInterval（）
    • setInterval和setTimeout的用法完全一致，区别仅仅在于setInterval指定某个任务每隔一段时间就执行一次，也就是无限次的定时执行。
  • clearTimeout（）
    • 将setTimeout返回的整数值（表示计数器编号）传入clearTimeout函数，取消对应的定时器。
  • clearInterval（）
    • 将setInterval返回的整数值（表示计数器编号）传入clearInterval函数，取消对应的循定时器。
  • 定时器运行机制
    • 定时器的运行机制，是将指定的代码移出本轮事件循环，等到下一轮事件循环，再检查是否到了指定时间。如果到了，就执行对应的代码；如果不到，就继续等待。
    • 这意味着，setTimeout和setInterval指定的回调函数，必须等到本轮事件循环的所有同步任务都执行完，才会执行。由于前面的任务到底需要多少时间执行完，是不确定的，所以没有办法保证，setTimeout和setInterval指定的任务，一定会按照预定时间执行。
  • 注意点：
    • setTimeout的作用是将代码推迟到指定时间执行，如果指定时间为0，即setTimeout(f,0),是不会立即执行指定的回调函数f。必须要等到当前脚本的同步任务，全部执行完毕以后，才会执行。

    • ‘==’ 和‘===’的区别

      第一种是==比较，它会自动转换数据类型再比较，很多时候，会得到非常诡异的结果；

      第二种是===比较，它不会自动转换数据类型，如果数据类型不一致，返回false，如果一致，再比较。

      由于JavaScript这个设计缺陷，不要使用==比较，始终坚持使用===比较。
